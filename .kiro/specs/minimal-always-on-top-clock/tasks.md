# Implementation Plan

- [ ] 1. アプリケーション基盤とデスクトップ実行環境を構築する
- [x] 1.1 複数画面を提供するフロントエンドと自動テストの基盤を構成する
  - 時計画面と設定画面を個別に起動・ビルドできる React/TypeScript の複数エントリ基盤を準備する。
  - 厳格な型検査と UI およびサービスの自動テストを実行できる開発構成を追加する。
  - 完了時に、空の時計画面と設定画面を含むフロントエンドをビルドでき、テストランナーを起動できる。
  - _Requirements: 2.1, 5.1_
- [x] 1.2 Windows デスクトップシェルとネイティブ機能の実行基盤を構成する
  - Windows 向けデスクトップアプリを開発・ビルドできる Tauri/Rust の実行基盤を作成する。
  - トレイ常駐、ローカル設定保存、および限定されたネイティブ操作を組み込めるホストを準備する。
  - 完了時に、デスクトップシェルが起動し、トレイおよび設定保存機能を登録可能なネイティブ基盤をビルドできる。
  - _Requirements: 1.1, 5.1, 7.1_
- [x] 1.3 時計ウィンドウと設定ウィンドウのホスト構成および許可境界を整備する
  - 時計ウィンドウを初期非表示、枠なし、透明、固定サイズ、タスクバー非表示で扱うための静的設定を準備する。
  - 時計画面と設定画面へ必要最小限のイベントおよび許可済み操作だけを付与し、Store の直接操作、不要なファイルシステム、外部通信権限を含めない。
  - 時計およびトレイで使用するアプリ資産をホスト構成に組み込む。
  - 完了時に、透明時計、設定画面、トレイ、および保存設定を最小権限で起動可能なホスト構成が得られる。
  - _Requirements: 1.2, 5.1, 7.1_

- [ ] 2. 設定契約と時計表示の中核機能を実装する
- [x] 2.1 表示設定と画面間通知の型付き契約を定義する
  - 表示モード、秒表示、時間表記、日付表示、最前面状態を含む完全な設定スナップショットと既定値を定義する。
  - 適用済み設定と `saved` または `volatile` の永続化状態を時計画面と設定画面の双方へ通知し、時計ウィンドウ可視状態を通知する payload およびイベント識別を定義する。
  - 後続の表示、保存、ネイティブ通知が同一の契約に依存できるよう、設定値の妥当性判定を提供する。
  - 完了時に、設定スナップショットと二種類の画面間通知を型検査可能な共有契約として利用できる。
  - _Requirements: 3.2, 3.3, 5.3, 5.4, 5.5, 6.1, 7.3, 8.2_
- [x] 2.2 (P) Rust ランタイム向けの設定保存、復元、および既定値復旧を実装する
  - Rust 側の永続化境界で設定スナップショットを保存し、次回起動時に検証済みの値として復元する。
  - 保存値が存在しない、不正である、または保存領域を利用できない場合は既定値で起動を継続する。
  - 保存に成功した完全な設定値をランタイムの冷間起動候補として返し、適用後の保存失敗に伴う実行中状態はランタイムが区別して扱えるようにする。
  - 完了時に、設定値を保存して再読込でき、異常時は定義済みの初期表示へ復旧する。
  - _Depends: 1.3, 2.1_
  - _Requirements: 6.1, 6.2, 6.3, 6.4_
  - _Boundary: SettingsStore_
- [x] 2.3 (P) 表示状態に応じた時計更新スケジューリングを実装する
  - Windows が提供する現在日時を表示更新へ渡す更新処理を用意する。
  - 秒表示時は秒境界、秒非表示時は分境界だけで更新を通知する。
  - 時計が非表示の間は更新通知を停止し、再表示時に現在日時へ同期できるようにする。
  - 完了時に、可視状態と設定に応じて不要な tick が発生せず、必要な境界でのみ表示用日時が更新される。
  - _Depends: 1.1, 2.1_
  - _Requirements: 2.1, 2.2, 2.3, 8.1, 8.2_
  - _Boundary: ClockScheduler_
- [x] 2.4 (P) デジタル表示とアナログ表示を実装する
  - 数値時刻を 12 時間または 24 時間形式で表示し、設定された場合のみ日付を併記する。
  - 文字盤と時針および分針を描画し、設定された場合のみ秒針を表示する。
  - 表示部品は受け取った現在日時と共有された設定契約のみから描画結果を決定する。
  - 完了時に、固定日時の表示をデジタルとアナログで切り替えた際、時間表記、日付、および秒針の有無が設定どおり表示される。
  - _Depends: 1.1, 2.1_
  - _Requirements: 2.4, 2.5, 3.1, 3.2, 3.3, 3.4, 3.5, 3.6_
  - _Boundary: DigitalClock, AnalogClock_
- [x] 2.5 (P) 時計の小型半透明表示面とドラッグ操作領域を実装する
  - 作業領域を占有しない固定サイズの時計表示面へ、半透明で読み取れる視覚表現を適用する。
  - 枠なしウィンドウ上で時計表示を保持したまま利用者がドラッグ移動できる操作領域を提供する。
  - 完了時に、時計画面が小型の半透明表示として描画され、ドラッグ可能な表示面を備える。
  - _Depends: 1.1, 1.3_
  - _Requirements: 1.2, 1.4_
  - _Boundary: clock presentation styles_

- [ ] 3. ネイティブウィンドウ、適用設定状態、および常駐ライフサイクルを実装する
- [x] 3.1 時計ウィンドウの初期配置、最前面、および非表示化を実装する
  - ランタイムが冷間起動時に保存設定または既定値を読み込み、適用済み状態を初期化した後に、プライマリ作業領域の右上付近へ時計を配置して最前面状態を適用し表示する。
  - 時計で `Alt+F4` により発生するクローズ操作はプロセス終了ではなく非表示へ変換し、共有された可視状態通知を操作完了後に発行する。
  - 配置情報を取得できない場合もフォールバック位置で時計を表示可能にする。
  - 完了時に、初期化した時計が指定位置と最前面状態で表示され、閉じても常駐状態のまま非表示になる。
  - _Depends: 1.3, 2.1_
  - _Requirements: 1.1, 1.3, 7.2, 8.2_
- [x] 3.2 設定ウィンドウ、トレイ操作、および明示終了を実装する
  - 設定ウィンドウの起動要求は既存ウィンドウの再利用またはフォーカスに収束させる。
  - トレイから時計の再表示、設定表示、および終了操作を提供し、再表示時に共有された可視状態通知を発行する。
  - 明示終了時のみ時計、設定、トレイ、およびアプリプロセスを終了する。
  - 完了時に、時計を閉じた後もトレイから再表示と設定表示ができ、終了操作によってのみプロセスが終了する。
  - _Depends: 2.1, 3.1_
  - _Requirements: 5.1, 5.2, 7.1, 7.3, 7.4, 7.5_
- [x] 3.3 UI へ公開するデスクトップ操作のアダプターを実装する
  - ランタイム初期化、設定表示、設定適用、適用済み設定の取得、永続化再試行、終了の各許可済み操作を、画面側が一貫した失敗処理で要求できるようにする。
  - ランタイムが返す `saved` または `volatile` の適用済みスナップショットだけを画面状態として扱える呼出境界を維持する。
  - 完了時に、時計および設定画面が定義済み操作だけを呼び出せ、未保存の適用状態もランタイムから取得して表示できる。
  - _Depends: 3.1, 3.2_
  - _Requirements: 4.3, 5.1, 5.5, 7.5_

- [ ] 4. ユーザー操作と画面間同期を結合する
- [x] 4.1 設定変更および表示状態のイベント同期を接続する
  - 設定画面と時計メニューからの設定変更をランタイムの共通適用経路へ渡し、適用済み設定の変更通知を時計画面と設定画面の双方へ配信する。
  - 最前面状態の変更はランタイムでネイティブ適用が成功した場合のみ保存へ進め、保存に失敗した場合はランタイムが適用中状態を `volatile` として保持・通知し、永続化再試行を可能にする。
  - ランタイムが実行中の適用済み設定を一意に保持し、時計の復帰や設定画面の再表示では `volatile` を古い保存値で上書きしない。
  - 設定画面が未保存編集を保持中に時計メニューから最前面変更を受けた場合、最前面値と適用済み基準値だけを同期して他の編集値を保持する。
  - 時計の非表示および再表示通知を表示制御へ伝える同期経路を接続する。
  - 画面の開始時に購読し終了時に解除することで、重複通知や終了後更新を防ぐ。
  - 完了時に、どちらの変更入口を利用しても時計と設定画面が同一の適用済み設定へ収束し、非表示または復帰後の時計も正しい表示状態へ更新される。
  - _Depends: 2.1, 2.2, 2.3, 3.1, 3.2_
  - _Requirements: 5.4, 5.5, 7.3, 8.2_
- [x] 4.2 時計画面の初期化、描画、および復帰同期を統合する
  - ランタイムから初期 applied snapshot の通知を受けた後に時計表示を開始し、現在日時の更新と選択された時計表示を結合する。
  - 小型半透明のドラッグ可能な表示面で、設定変更通知に基づく表示と最前面状態を直ちに反映する。
  - 非表示中は視覚更新を停止し、トレイからの復帰時は実行中の適用済み設定を優先して現在日時と再同期する。
  - 完了時に、起動、設定変更、非表示、再表示の各操作で時計表示と最前面状態が実際の設定に一致する。
  - _Depends: 2.2, 2.3, 2.4, 2.5, 3.1, 3.3, 4.1_
  - _Requirements: 1.2, 1.3, 1.4, 2.1, 2.2, 2.3, 2.4, 2.5, 3.1, 3.2, 3.3, 3.4, 3.5, 3.6, 5.4, 5.5, 6.3, 7.3, 8.1, 8.2_
- [x] 4.3 設定画面の編集、保存、および即時通知を統合する
  - 表示モード、秒表示、時間表記、日付表示、および最前面状態を編集できる設定画面を提供する。
  - 設定確定時はランタイムの設定適用経路を使用し、通常の保存失敗時は未適用状態を示し、最前面適用後の保存失敗時は適用中だが未保存の状態と再試行操作を利用者へ示す。
  - 設定画面の再表示時はランタイムの適用済み設定を読み込み、未保存の最前面適用を古い永続値へ巻き戻さない。
  - 最前面状態の保存変更は表示中の時計へ直ちに適用されるフローへ接続する。
  - 完了時に、設定を確定すると時計の表示と最前面状態が即時更新され、再起動後も保存値が復元される。
  - _Depends: 2.2, 3.3, 4.1, 4.2_
  - _Requirements: 5.3, 5.4, 5.5, 6.2, 6.3_
- [x] 4.4 時計の操作メニューを設定、最前面切替、および終了フローへ統合する
  - 時計上の右クリックではブラウザ既定メニューを抑止し、設定、最前面切替、終了の操作だけを表示する。
  - 最前面項目には適用中の状態を示し、切替結果をランタイムの設定適用経路へ渡して、ネイティブ状態および開いている設定画面と整合させる。
  - 設定表示と終了は既存のデスクトップ寿命管理へ委譲する。
  - 完了時に、右クリック操作だけで設定表示、最前面状態の確認と切替、完全終了を実行できる。
  - _Depends: 2.2, 3.2, 3.3, 4.2_
  - _Requirements: 4.1, 4.2, 4.3, 4.4, 5.1, 7.5_
