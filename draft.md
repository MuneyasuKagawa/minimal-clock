# デスクトップ時計アプリ 実装引き継ぎ資料

## 0. このドキュメントについて

Tauri v2 で「画面隅に常駐する小型の時計アプリ」を実装するための要件・設計をまとめたもの。
Claude Code はこのドキュメントを起点に実装を進めること。実装の細部（ファイル分割の粒度、ヘルパ関数など）は任せるが、**機能要件・非機能要件・技術選定は本書に従うこと**。

---

## 1. プロダクト概要

- 画面の隅にちょこんと常駐する、小さなデスクトップ時計。
- タイトルバー・ウィンドウ枠なし。背景は半透明で、デスクトップに溶け込む見た目。
- 時計ウィンドウ自体を右クリックするとコンテキストメニューが出る。
- メニューから「設定」を開くと、別ウィンドウで設定画面が表示される。
- システムトレイ（タスクトレイ／メニューバー）に常駐する。ウィンドウを閉じてもプロセスは終了せず、トレイから復帰・終了できる。

---

## 2. 技術スタック

| 項目 | 採用技術 | 補足 |
|------|----------|------|
| アプリフレームワーク | **Tauri v2** | v1ではない。API・設定ファイル構造が異なるため必ずv2系で。 |
| バックエンド | Rust | Tauri標準。最小限のコマンドのみ実装。 |
| フロントエンド | TypeScript + React | UIはWeb技術で構築。ビルドはVite想定。 |
| スタイリング | CSS（またはTailwind） | 実装者判断でよい。半透明・影など軽量に作れる方を選ぶ。 |
| アナログ時計描画 | **SVG**（推奨） | 軽量・スケーラブル・CSSで調整しやすい。最終判断は実装側に委ねるが、特段の理由がなければSVGで。 |
| 設定の永続化 | `tauri-plugin-store` | JSONで設定値を保存。下記「設定の永続化」参照。 |

### ターゲットプラットフォーム

- **第一ターゲット: Windows**。まずWindowsで完全動作させる。
- macOS は将来対応の可能性あり。コードはなるべくOS依存を局所化しておく。
- Linux は当面対象外（WM依存で挙動が割れやすいため）。MVPに含めない。

---

## 3. 機能要件

### 3.1 時計表示

- 現在時刻をリアルタイム更新で表示する。
- 表示モードは **デジタル / アナログ** を設定で切り替え可能。
- **秒の表示・非表示** を設定で切り替え可能（デジタル・アナログ両モードに適用）。
  - デジタル: 秒なしなら `HH:MM`、秒ありなら `HH:MM:SS`。
  - アナログ: 秒なしなら秒針を描画しない、秒ありなら秒針を描画。
- **12時間 / 24時間表記** を設定で切り替え可能（デジタル表示に適用。アナログは文字盤の都合上12時間固定でよい）。
- **日付表示の有無** を設定で切り替え可能。

> 更新間隔: 秒表示ありのときは1秒ごと。秒表示なしのときは負荷低減のため分単位の更新でよい（実装者判断）。アナログの秒針はなめらかに動かす必要はなく、1秒刻みのカチカチ更新で構わない。

### 3.2 コンテキストメニュー（時計ウィンドウ上で右クリック）

最低限、以下の項目を含む。

- **設定** … 設定ウィンドウを開く。
- **常に最前面** … トグル（ON/OFFのチェック状態を反映）。
- **終了** … アプリ（プロセス）を完全終了する。

> ネイティブのコンテキストメニューでもWeb側で描画するメニューでもよいが、ウィンドウが枠なし・小型のため**Web側でカスタム描画するのが扱いやすい**と思われる。実装者判断。OSのデフォルト右クリックメニュー（リロード等）は抑制すること。

### 3.3 設定ウィンドウ

別ウィンドウ（通常の枠あり・タイトルバーありでよい）で開く。設定項目:

- 表示モード: デジタル / アナログ（ラジオまたはトグル）
- 秒の表示: ON / OFF
- 時間表記: 12時間 / 24時間
- 日付表示: ON / OFF
- 常に最前面: ON / OFF
- （任意）時計のサイズ、文字色・背景透過度などの外観調整。MVPでは必須ではないが、設定構造に拡張余地を残しておくこと。

設定変更は**即座に時計ウィンドウへ反映**されること（保存ボタンを押した時点、または変更時リアルタイムのどちらでもよいが、挙動を一貫させる）。

### 3.4 システムトレイ常駐

- トレイアイコンを常駐させる。
- トレイアイコンのメニューにも最低限「設定」「終了」を用意する。
- **ウィンドウのクローズ操作ではプロセスを終了させない**（時計ウィンドウを隠す／トレイに格納する挙動）。完全終了はメニューの「終了」からのみ。

---

## 4. 非機能要件

- **軽量であること**。これが最優先事項のひとつ。常駐メモリを無駄に消費しない。重いポーリングや過剰な再描画を避ける。
- 起動が速いこと。
- 枠なし・最前面・半透明ウィンドウを正しく実現すること。

---

## 5. ウィンドウ仕様（tauri.conf.json の要点）

### 時計ウィンドウ（メイン）

- `decorations: false`（タイトルバー・枠なし）
- `transparent: true`（背景透過。CSS側で半透明背景を描く）
- `alwaysOnTop: true`（初期値。設定でトグル可能にする）
- `resizable: false`
- `skipTaskbar: true`（タスクバーに出さない。常駐小物のため）
- `shadow: false`（透過と相性が悪い場合があるため要調整）
- 初期サイズは時計が収まる小さめの固定サイズ。
- 起動時の初期位置は**画面右上隅**に配置する。マルチモニタや解像度差を考慮し、プライマリモニタの作業領域を基準に右上へ寄せる処理をRust側またはJS側で行う。
- ウィンドウは**ドラッグで移動可能**にする（枠がないため、時計領域のドラッグでウィンドウを動かせるよう `data-tauri-drag-region` 等で実装）。移動後の位置は記憶して次回起動時に復元するのが望ましい（MVPでは必須ではない）。

> 注意: `transparent: true` はWindowsで `decorations: false` とセットで使う前提。透過まわりはOSごとの差異があるため、まずWindowsで確実に動かす。

### 設定ウィンドウ

- 通常の装飾あり（タイトルバーあり）。
- 起動時には開かない。メニューから開いたときに生成・表示する。
- 二重に開かないよう、既に開いていればフォーカスを当てるだけにする。

## 6. 設定の永続化

- `tauri-plugin-store` を用いてJSONで保存する。
- 保存する設定値の型（TypeScript側のイメージ）:

```ts
type ClockSettings = {
  mode: "digital" | "analog";
  showSeconds: boolean;
  hour24: boolean;       // true=24時間表記, false=12時間表記
  showDate: boolean;
  alwaysOnTop: boolean;
  // 拡張余地（MVPでは未使用でも型に含めてよい）
  // size?: number;
  // opacity?: number;
  // position?: { x: number; y: number };
};
```

- 初期値（デフォルト）の例: `mode: "digital"`, `showSeconds: true`, `hour24: true`, `showDate: false`, `alwaysOnTop: true`。
- 起動時に保存値を読み込み、なければデフォルトで初期化する。
- 設定ウィンドウと時計ウィンドウは別ウィンドウなので、**設定変更をウィンドウ間で同期**する必要がある。Tauriのイベント（`emit`/`listen`）で「設定が変わった」通知を飛ばし、時計ウィンドウ側が再読み込み・再描画する構成を推奨。

## 7. 推奨ディレクトリ構成（目安）

実装者が調整してよいが、おおよそ以下を想定。

```
clock-app/
├── src/                          # フロントエンド (React + TS)
│   ├── main.tsx                  # エントリ
│   ├── clock/                    # 時計ウィンドウ用
│   │   ├── ClockApp.tsx
│   │   ├── DigitalClock.tsx
│   │   ├── AnalogClock.tsx       # SVGで描画
│   │   └── ContextMenu.tsx
│   ├── settings/                 # 設定ウィンドウ用
│   │   └── SettingsApp.tsx
│   ├── lib/
│   │   ├── settings.ts           # store読み書き・型定義・デフォルト
│   │   └── events.ts             # ウィンドウ間イベント名の定数等
│   └── styles/
├── src-tauri/
│   ├── src/
│   │   ├── main.rs
│   │   └── lib.rs                # トレイ・ウィンドウ生成・コマンド
│   ├── tauri.conf.json
│   ├── Cargo.toml
│   └── icons/

├── settings.html                 # 設定ウィンドウ（別エントリ）
```

## 8. Rust側で必要になる主な実装

- システムトレイ（`TrayIconBuilder`）とそのメニュー（設定・終了）。
- 設定ウィンドウを開くコマンド（既に開いていればフォーカス）。
- 「常に最前面」トグルを反映する処理（`set_always_on_top`）。
- ウィンドウのクローズ要求をインターセプトし、終了ではなく非表示／トレイ格納にする処理。
- 起動時に時計ウィンドウを右上隅へ配置する処理（モニタ作業領域取得 → 座標計算）。
- 必要なら設定変更イベントの中継。

## 9. MVPの完成条件（受け入れ基準）

以下がWindowsで動作すれば第一段階完了とみなす。

1. 起動すると画面右上隅に、枠なし・半透明の小さな時計が表示される。
2. 時計はリアルタイムに更新される。
3. 時計を右クリックするとコンテキストメニューが出て、「設定」「常に最前面」「終了」が選べる。
4. 「設定」で別ウィンドウが開き、デジタル/アナログ・秒表示・12/24時間・日付表示・最前面を変更できる。
5. 設定変更が時計に即時反映される。
6. 設定はアプリ再起動後も保持される。
7. トレイアイコンが常駐し、ウィンドウを閉じてもプロセスは生き続け、トレイ／メニューの「終了」でのみ完全終了する。
8. アナログ時計はSVGで描画され、秒表示OFF時は秒針が消える。

---

## 10. 留意点・既知の落とし穴

- **透過 + 枠なし**はWindowsで動作確認を最優先に。`transparent` と `decorations` の組み合わせ、影（shadow）の扱いはOS差があるので、見た目が崩れたらまず影を切って確認する。
- 枠なしウィンドウは**ドラッグ移動の実装を忘れない**（`data-tauri-drag-region`）。ただしドラッグ領域と右クリック・クリック操作が干渉しないよう注意。
- 設定ウィンドウと時計ウィンドウは**別プロセスではなく別ウィンドウ**だが、JSのstateは共有されない。設定同期はstore＋イベントで行う。
- 秒表示OFF時に毎秒再描画しない（軽量性のため）。
- Tauri v2 はプラグインの導入方法・権限（capabilities/permissions）設定がv1と異なる。`tauri-plugin-store` などは v2 対応版を使い、capabilities に必要な権限を明示すること。
- macOS将来対応に備え、トレイ常駐やウィンドウ挙動のOS依存コードはなるべくまとめておく。
