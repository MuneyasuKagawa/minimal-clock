# Minimal Clock

Небольшое приложение-часы, всегда отображаемое поверх других окон, для рабочего стола Windows, которое позволяет следить за текущим временем во время работы. Оно держит полупрозрачные часы поверх остальных окон, позволяет менять отображаемое содержимое в зависимости от ваших нужд и располагается в системном трее, поэтому продолжает работать, пока вы явно не закроете его.

- Компактное полупрозрачное окно без рамки и заголовка
- Переключатель «поверх всех окон» (Always on Top)
- Цифровой и аналоговый режимы отображения (простой / с цифрами / с метками)
- 12 / 24-часовой формат, отображение секунд, мигающее двоеточие и необязательная дата
- Постоянное присутствие в системном трее — закрытие окна не завершает работу приложения
- Настройки сохраняются локально и восстанавливаются при следующем запуске

## Технологический стек

| Слой | Технология |
| --- | --- |
| Оболочка рабочего стола | [Tauri 2](https://tauri.app/) (Rust) |
| Фронтенд | React 19 + TypeScript |
| Сборка / сервер разработки | Vite 7 |
| Тесты | Vitest + Testing Library (фронтенд), `cargo test` (Rust) |
| Сохранение настроек | `tauri-plugin-store` |
| Запуск задач | [mise](https://mise.jdx.dev/) |

Это многооконная конфигурация с двумя точками входа WebView: окно часов (`index.html`) и окно настроек (`settings.html`).

## Требования

- Node.js (версия, поставляемая с `npm`)
- Инструментарий Rust (`cargo`) — необходим для сборки Tauri
- Windows (целевая платформа этого приложения)
- [Предварительные требования Tauri 2](https://tauri.app/start/prerequisites/) (WebView2 и т. д.)

## Установка

```bash
npm install
```

## Разработка

С помощью mise:

```bash
mise run dev             # Запустить десктопное приложение Tauri
mise run frontend:dev    # Только фронтенд (сервер разработки Vite)
```

Запуск скриптов npm напрямую:

```bash
npm run tauri:dev   # Запустить десктопное приложение Tauri
npm run dev         # Только сервер разработки Vite (http://localhost:1420)
```

## Сборка

```bash
mise run build        # Собрать десктопное приложение Tauri
# или
npm run tauri:build   # То же самое
npm run build         # Только фронтенд (проверка типов + сборка Vite)
```

## Тесты и проверка типов

```bash
npm test          # Vitest (фронтенд)
npm run typecheck # Проверка типов с помощью tsc
cargo test        # Тесты на стороне Rust (запускать в каталоге src-tauri)
```

## Обзор архитектуры

Фронтенд (React) и бэкенд (Rust/Tauri) взаимодействуют через команды и события Tauri.

- **Команды Tauri**: `initialize_clock_window`, `open_settings_window`, `apply_settings`, `get_applied_settings`, `retry_settings_persistence`, `quit_application`, `native_runtime_capabilities`
- **События**: изменения настроек (`settings-changed`) и состояние видимости окна часов транслируются между окнами, поэтому изменения, внесённые в окне настроек, немедленно отражаются в окне часов.
- **Сохранение**: `apply_settings` сохраняет настройки в хранилище; если сохранение не удалось, настройки остаются в энергозависимом состоянии (`volatile`) и могут быть повторно сохранены через `retry_settings_persistence`.
- **Присутствие в трее**: закрытие окна часов только скрывает его — процесс продолжает работать, и вы можете использовать «Настройки», «Восстановить» и «Выход» из трея.

Подробные требования и проектирование см. в `.kiro/specs/minimal-always-on-top-clock/`.
